Skip Trace API

The Skip Trace API allows you to create skip trace tasks and retrieve their status. All requests must include a valid JWT token in the Authorization header.

Base URL:
https://landportal.com/wp-json/lp-rest-api/v1

Authentication

All requests must include a JWT token:

Authorization: Bearer <JWT_TOKEN>

1. Create Task

Endpoint: POST /skip-trace
Headers:
Content-Type: application/json
Authorization: Bearer <JWT_TOKEN>
Required Conditions:
  • Active subscription
  • Sufficient tokens
  • Either input_file or json_data must be provided
  • When using input_file, a fields mapping must be included

Option 1: File via URL

{
  "method": "create",
  "input_file": "https://example.com/data.csv",
  "has_header": true,
  "fields": {
    "last_name": "last_name",
    "first_name": "first_name",
    "mailing_address": "mailing_address",
    "mailing_city": "mailing_city",
    "mailing_state": "mailing_state",
    "mailing_zip": "mailing_zip",
    "property_address": "property_address",
    "property_city": "property_city",
    "property_state": "property_state",
    "property_zip": "property_zip"
  },
  "emails_trace": false,
  "test_mode": "true"
}

Option 2: Inline JSON Data

{
  "method": "create",
  "json_data": [
    {
      "last_name": "Smith",
      "first_name": "John",
      "mailing_address": "123 Main St",
      "mailing_city": "New York",
      "mailing_state": "NY",
      "mailing_zip": "10001",
      "property_address": "456 Oak Ave",
      "property_city": "Brooklyn",
      "property_state": "NY",
      "property_zip": "11201"
    }
  ],
  "emails_trace": false,
  "test_mode": "true"
}

Success Response

{
  "success": true,
  "message": "Skip trace created successfully",
  "data": {
    "data_length": 100,
    "input_file": "https://s3.amazonaws.com/bucket/skip-trace-files/uuid_userid_timestamp.csv",
    "task_id": 12345,
    "tokens_left": 950
  }
}

Test Mode Parameter

test_mode: Optional parameter that can be set to "true" for testing purposes.
Test Mode Behavior:
  • Fake Data: When test_mode: "true" is set, the output file will be populated with fake/sample data instead of real skip trace results
  • No Token Usage: Tokens are not consumed when running in test mode, allowing you to test the API without affecting your token balance
  • Development Testing: Perfect for testing your integration, data processing logic, and API responses without using real data or tokens
  • Response Format: The response structure remains the same, but the actual data in the output file will be sample data

Examples

cURL
curl -X POST "https://landportal.com/wp-json/lp-rest-api/v1/skip-trace" \
  -H "Authorization: Bearer <JWT_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "method": "create",
    "json_data": [{"last_name":"Smith","first_name":"John"}],
    "test_mode": "true"
  }'
Python (requests)
import requests

url = "https://landportal.com/wp-json/lp-rest-api/v1/skip-trace"
headers = {"Authorization": "Bearer <JWT_TOKEN>"}
payload = {
    "method": "create",
    "json_data": [{"last_name": "Smith", "first_name": "John"}],
    "test_mode": "true"
}

response = requests.post(url, json=payload, headers=headers)
print(response.json())
JavaScript (fetch)
const response = await fetch("https://landportal.com/wp-json/lp-rest-api/v1/skip-trace", {
  method: "POST",
  headers: {
    "Authorization": "Bearer <JWT_TOKEN>",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    method: "create",
    json_data: [{ last_name: "Smith", first_name: "John" }],
    test_mode: "true"
  })
});

const data = await response.json();
console.log(data);

3. Stop Task

Endpoint: POST /skip-trace
Headers:
Content-Type: application/json
Authorization: Bearer <JWT_TOKEN>
Required Parameters:
  • method - Must be "stop"
  • task_id - The ID of the task to stop
Note: This endpoint allows you to stop a running skip trace task. Once stopped, the task cannot be resumed and any partial results will be lost.

Request Body

{
  "method": "stop",
  "task_id": 12345
}

Success Response

{
  "success": true,
  "message": "Skip trace task stopped successfully",
  "data": {
    "response_code": 200
  }
}

Examples

cURL
curl --location 'https://landportal.com/wp-json/lp-rest-api/v1/skip-trace' \
--header 'Authorization: Bearer <JWT_TOKEN>' \
--header 'Content-Type: application/json' \
--data '{
    "method":"stop",
    "task_id":12345
}'
Python (requests)
import requests

url = "https://landportal.com/wp-json/lp-rest-api/v1/skip-trace"
headers = {
    "Authorization": "Bearer <JWT_TOKEN>",
    "Content-Type": "application/json"
}
payload = {
    "method": "stop",
    "task_id": 12345
}

print(response.json())

  method: "POST",
  headers: {
    "Authorization": "Bearer <JWT_TOKEN>",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    method: "stop",
    task_id: 12345
  })
});

const data = await response.json();
console.log(data);

4. Continue Task

Endpoint: POST /skip-trace
Headers:
Content-Type: application/json
Authorization: Bearer <JWT_TOKEN>
Required Parameters:
  • method - Must be "continue"
  • task_id - The ID of the task to continue
Note: This endpoint allows you to resume a previously stopped skip trace task. The task will continue from where it was stopped and will be assigned a new Celery task ID for processing.

Request Body

{
  "method": "continue",
  "task_id": 12345

Success Response

{

Errors

401 Unauthorized

{ "success": false, "message": "User not authenticated" }

400 Validation

{ "success": false, "message": "Input file or JSON data is required" }

403 Forbidden

{ "success": false, "message": "You are not authorized to get this skip trace" }

500 Server Error

{ "success": false, "message": "Internal server error occurred" }

File Handling

  • Supported format: CSV
  • Encoding: UTF-8
  • Max size: Limited by available tokens
  • URLs supported: Direct links, Google Drive links

Token System

  • 1 row = 1 token
  • Tokens are deducted on task creation
  • Failed rows refund tokens

Comp Reports API

The Comp Reports API allows you to generate and retrieve comp reports for a given property. You can create new reports, update existing ones, and retrieve completed reports with detailed property analysis.

Base URL:
https://landportal.com/wp-json/lp-rest-api/v1

  • Authentication
  • Create Comp Report
  • Update Comp Report
  • Get Comp Report

Authentication

All requests must include a JWT token:

Authorization: Bearer <JWT_TOKEN>

Create Comp Report

Endpoint: POST /reports
Headers:
Content-Type: application/json
Authorization: Bearer <JWT_TOKEN>
Required Parameters:
  • method - Must be "create_comp_report"
  • propertyid - Property ID from landportal.com
  • fips - FIPS code for the property location
Note: Comp report creation is an asynchronous process. The API will return immediately with a confirmation that the request has been sent. Use the "Get Comp Report Status" endpoint to check the progress and retrieve the completed report.

Request Body

{
  "method": "create_comp_report",
  "propertyid": "12345678",
  "fips": "12345"
}

Success Response

{
  "success": true,
  "message": "Comp report generation request sent",
  "data": {
    "property_id": "12345678",
    "fips": "12345"
  }
}
